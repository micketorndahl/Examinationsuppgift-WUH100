Processanalys

Hur går jag till väga när jag skapar en hemsida?

Jag börjar med att planera, vad är viktigt för just denna sida? Vart ska fokuset ligga? När jag bestämt mig för vad som är viktigt börjar jag tänka på strukturen, hur ska de olika delarna fördelas på sidan, hur många sidor behöver jag? Här kan skisser/blueprints/mockups/framewires vara en god idé för att lättare få en översikt och inte förhastigt börja arbeta med uppmärkning/kod, det kan lätt bli arbete i blindo då. Ett annat alternativ är det som Christian Olsson lärare i kursen WUH100 (Webbutveckling HTML / CSS) nämner i sin "Tips Trix 01" video, Christian inleder: "En whiteboard är ett fantastiskt bra verktyg att göra layoutskisser med, det går så snabbt att komma ifrån en idé till en nedritad design" (WUH100 2018 TipsTrix 01 00:13:15)

Verktygen för att skapa en hemsida

Nu är det dags att komma igång med det skarpa arbetet. Jag vill märka upp, stila och fylla dokumenten med innehåll för att det slutgiltigen ska utgöra en webbplats/hemsida.

HTML

Vi börjar med HTML med denna förklaring från wikipedia:

HTML ger möjlighet att ange ett dokuments struktur (rubriker, styckeindelning m.m.), metainformation (språk, författare, plats i en hierarki) och i viss mån hur dokumentet skall visas. HTML tillåter också att man infogar information av andra typer, eventuellt i andra filer, från början framförallt bilder, senare till exempel datorprogram (java, javascript) och instruktioner för hur sidan skall visas (CSS). (Wikipedia, 2021)

Jag skapar ett par stycken .html-filer och varje fil representerar en delsida av hemsidan. index.html är startfilen, det är den fil vi utgår ifrån och den som innehåller välkomstsidan och ofta huvudinnehållet (Main-Content). De andra döps efter det dem ska innehålla (About, contact etc)

Man kan ofta fylla index.html med en hel del och sen copy/pastea det vidare till de andra sidorna så att det får samma grundmall för att sedan specifiera dem för ändamålet de har.

Jag börjar med att skapa en huvudsektion som är basen för huvudsidan index.html
Jag använder mig av det semantiska HTML elementen <main> Här kommer en förklaring till semantisk HTML.

Semantic HTML, also known as semantic markup, refers to the use of HTML tags that convey the meaning—or semantics—of the content contained within them. By adding semantic HTML tags to your pages, you provide additional information that helps define the roles and relative importance of the different parts of your page. As opposed to non-semantic HTML, which uses tags that don’t directly convey meaning.
- Vlado Pavlik (Semrush Blog, 2022)

Semantiska element är förklarande och underlättar för personer med hjälpmedel som t.e.x läsplattor. Det hjälper även webbutvecklare att navigera runt lättare när elementen beskriver innehållet.

Det är i elementet main som jag kommer fylla med text för den här processanalysen kring att skapa en hemsida. Eftersom det är mycket text delar jag upp processanalysen i delar(sektioner) därmed använder jag mig av det semantiska elementet <section>. Jag vet att jag vill ha 3 kolumner/sektioner för att dela upp innehållet och göra det mer lättläsligt. Skulle texten löpa genom enbart en sektion från sida till sida på skärmen skulle det bli väldigt svårläst

Jag skapar 3 sektioner den här gången och kallar dem section1, 2 och 3. Detta för att senare kunna använda CSS och stila upp dessa till spalter bredvid varandra. Det är viktigt att sektionerna omslutas av main-elementet då det utgör en container(box/ram) som vi använder för att gruppera innehållet och stila det tillsammans med de CSS verktyg som finns. De olika sektionerna omsluter i sin tur dem rubriker och paragrafer jag vill ha i just den sektionen. Rubrik skrivs innanför <h1> - <h6> och du väljer från lågt - till högt beroende på hur viktiga dem är för innehållet (h1 är viktigast(heading 1) och sen går det nedåt i rang. paragrafer skrivs innanför elementet <p> och omslutar all brödtext.

Inom brödtexten (paragraferna) vill jag ibland referera till en källa eller citera någon och då använder jag mig av <q> för ett kort inline citat eller <blockquote> för ett som det låter block av text från en källa, det kan vara något beskrivande som styrker det man vill framföra. Här kommer exempel på båda:
Websites should look good from the inside and out.
– Paul Cookson (Austin, 2020) Ett bra citat såväl som uppmuntring? Det här blockcitatet träffar också rätt:

Learning HTML and CSS is a lot more challenging than it used to be. Responsive web design adds more layers of complexity to design and develop websites. – Jacob Lett (Austin, 2020)

Det är dags för att strukturera upp navigationen och headern samt stila dessa. Jag skapar en div container med namnet "top-container" som ska komma att innehålla både headern och navigationen (länkar till de olika sidorna på webbsidan) Header tagen som jag också ger ett id med namnet header för att kunna stila senare omfamnar h1 tagen för den största rubriken på webbsidan som i detta fall inehåller mitt namn samt infotexten "Portfolio". Jag gör h1 rubriken klickbar med länk till startsidan med en <a> tag med destination "index.html". Jag omfamnar ordet portfolio och ett "/" tecken med inline containern <span> för att kunna stila dessa annorlunda med CSS.

Vidare till navigationen. Jag använder mig av den semantiska(berättande) tagen <nav> (navigation) som i sin tur omfamnar en unordered list <ul> eftersom det i sin tur ska innehålla flera <li> tags som står för list items, inom dessa ska det vara en <a> tag för varje intern länk till de olika sidorna i navigationen. <a href="index.html"> etc. Jag avslutar alla tags med </element> och jag kommenterar över och under olika sektioner för att enklare kunna navigera sig igenom all uppmärkning/kod. Syntax för kommentar:
<!-- Sektion Start --> och sen likadant fast Sektion End när den sektionen är slut.

Efter detta är det dags för uppmärkning av bannern(baner/flagga) som är den del som ska fånga uppmärksamhet, ofta med en slogan i stor text. I mitt fall ska den innehålla en bakgrundsbild som beskriver mig och som fångar intresse. Jag skapar en section tag med id't banner. ID eftersom den används unikt en gång. Klass väljer jag vanligtvis när något ska användas flera gånger. Bannern kommer skapas genom CSS style med background-image så vi kommer till det senare men den ska innehålla 2 till "div" för att jag vill ha en bar i bannerns underkant som jag kan fylla med text.

Jag skapar två "divvar" (<div>) en för att skapa en container (jag färgar denna senare i gradient-style med css) och en innanför containern som ska innehålla några bilder med citat under.

I slutat av sidan skapar jag 2 <footer> sektioner som är ett semantiskt element för sidfot. Den första sidfoten är ämnad som en sidfot för referenserna hänvisade till processanalysen. Denna sidfot som är en div ges värdet(namnet) refs-footer (referenser) som i sin tur omslutar ul, li och a taggar för att ställa upp en lista och länka varje item (list-item) till källan (referensen). Jag använder <span> där jag vill kunna styla en del annorlunda

The <span> tag is an inline container used to mark up a part of a text, or a part of a document (W3schools.com, 2019)

Den slutgiltiga footern på sidan innehåller länkar till mina sociala medier och kontaktinfo med samma element som tidigare footer med lite extra css-klasser för ikoner som är inlänkade med font-awesome.

Font Awesome is the Internet's icon library and toolkit, used by millions of designers, developers, and content creators.t (Font Awesome 5, 2017)

CSS

Det är dags för att stila upp dokumentet, vi vill informera webbläsaren om hur det ska visas för användaren. Jag skapar en mapp som heter Styles där jag kan lägga flertal olika delar av stilandet i separata filer/stilnings-dokument för att jag ska slippa en stor/lång och svårläst fil.

Vad är då CSS? Här kommer en förklaring från wikipedia:

Cascading Style Sheets (CSS, på svenska stilmall) är ett språk som beskriver presentationsstilen för ett strukturerat dokument som till exempel typsnitt, textstorlek och färg. Tekniken är ett sätt att anpassa dokumentet med hänsyn till datortyp, skärmupplösning, färgdjup och installerade typsnitt.

Nu vidare till struktureringen av mina stilmallar. Jag inleder med en css-reset fil i min Styles/ mapp och i detta fall använder jag mig av Erics Meyer's, varför gör jag detta? Här kommer en förklaring från meyerweb.com

The goal of a reset stylesheet is to reduce browser inconsistencies in things like default line heights, margins and font sizes of headings, and so on. (Meyerweb.com, 2019)

Wikipedias förklarande:

A reset stylesheet (or CSS reset) is a collection of CSS rules used to clear the browser's default formatting of HTML elements, removing potential inconsistencies between different browsers. (Wikipedia, 2021)

Eftersom CSS teknik är baserat på kaskad (Cascading) så är det viktigt att tänka på vilken ordning stilandet implementeras (Tänk ett vattenfall) Vi börjar med det som är stommen och sen kommer det senare att fylla på eller skriva över det tidigare om man nu så vill (Det finns användning för det)

Jag skapar en fil i Styles mappen med namnet style.css detta kommer att vara den stilfel med allt huvudinnehåll för stilandet. I filen inleder jag med göra en "box-sizing-reset" med denna förklaring:

You might have been dealing with a lot of mathematical calculations to size up different elements, if you are not using this CSS box-sizing reset. You might already know about box-sizing CSS property, that lets you choose a sizing nature for the box. By default, it is set to content-box. The border-width and padding values when added to a content-box, are exclusively added to the whole width of the box. (W3Bits, 2014)

Innehållet på en webbsida är uppdelade i flera olika boxar som tillsammans utgör hela sidan men för att strukturera upp stilandet finns det olika standardbeteende man kan ställa in för hur webbläsaren beräknar storleken på dessa boxar. Ifall du väljer att börja med en box-sizing-reset på det sättet jag ställt in det, slipper du räkna på marginaler, paddings (det som ligger runt din box) detta ingår istället i den totala bredden av boxen

Jag fortsätter med att längst upp i style.css skapa(skriva in) en TOC (Table Of Content) detta simplifierar läsandet av min fil genom att beskriva de olika delarna i en punktlista

Jag vill nu stila upp de olika delarna av mitt innehåll i de olika dokumenten, jag utgår ifrån första sidan (index.html) och gör den så bra och användarvänlig som möjligt, då kan jag lätt fylla på i de andra sidorna sedan och ändra det som behöver passa mer specifikt på den individuella sidan.

Jag använder mig av :root elementet för att skapa variabler och i detta fall till färger som ska förekomma i flera olika delar av sidan/sidorna. För att inte behöva upprepa mig kan jag tilldela ett namn som t.e.x "body-text-color" till en färg och om vill jag byta den färgen gör jag det längst upp i dokumentet (:root) jag slipper då att gå in och ändra på varje specifierad färg. Förklaring till :root

The :root CSS pseudo-class matches the root element of a tree representing the document. In HTML, :root represents the <html> element and is identical to the selector html, except that its specificity is higher. (developer.mozilla.org, n.d.)

Jag väljer body elementet och stylar det med en background-image: linear-gradient och sätter 4 olika färger som bakgrunden ska skiftas emellan (detta ger en fin övegång i färger, det är bra om man väljer snarlika färger för en mjuk övergång). Jag sätter basfärg för sidan genom color och tilldelar den min variabel i : root (body-text-color). Overflow-x: hidden för att "klippa" ifall något svämmar över ut i kanterna annars kan en skroll-bar i sidled dyka upp.

Jag väljer ett antal element som jag vill ge samma font-family (typsnitt), storlek, line-height etc

Jag väljer mina headings tillsammans för att först ge dem ett liknande utseende och sedan separat (h1, h2, h3 etc.) för att ge dem storleksskillnader. Jag använder mig av clamp för att göra sidan mer responsiv. Om clamp:

The clamp() CSS function clamps a middle value within a range of values between a defined minimum bound and a maximum bound. The function takes three parameters: a minimum value, a preferred value, and a maximum allowed value. (developer.mozilla.org, n.d.)

Jag använder mig av pseudo-elementet "::after" med h3 selektorn. "h3::after" för att stila varje h3 tag med en bar som dividerar rubrikerna.

In CSS, ::after creates a pseudo-element that is the last child of the selected element. It is often used to add cosmetic content to an element with the content property. It is inline by default. (developer.mozilla.org, n.d.)
